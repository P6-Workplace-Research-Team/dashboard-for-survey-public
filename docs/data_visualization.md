# 데이터 시각화 디자인 시스템

> **범위**: 시각화 영역의 디자인 토큰, 컴포넌트 스펙, 렌더링 규칙, 데이터 집계·계산·예외 처리 규칙을 함께 다룬다.
> **연계**: 공통 디자인 토큰(폰트, 중립 색상, 반경, 그림자)은 `design_system.md`를 참조한다. 이 문서에는 시각화 영역 전용 적용 기준만 기술한다.
> **AI 작업 규칙**: 이 문서에 정의된 토큰과 컴포넌트만 사용한다. `design_system.md`와 이 문서에 없는 색상·크기·간격을 임의로 추가하지 않는다. 새 토큰이 필요하면 반드시 사용자에게 확인을 요청한다.

## 0. 기본 개념

### 분석 대상 문항
- 결과를 직접 시각화해서 보여주는 중심 문항이다.
- 결과 카드 제목은 기본적으로 코드북의 `question_label`을 사용한다.
- 질문 원문이 있으면 `Q. {question_full}` 형식으로 함께 표시한다.

### 분석 기준 문항
- 분석 대상 문항의 결과를 그룹별로 나누어 비교할 때 사용하는 문항이다.
- 현재 대시보드는 단일응답 계열 그룹 문항과 시스템 컬럼 `survey_year`를 기준 문항 후보로 사용한다.
- 기준 문항을 선택하지 않으면 전체 결과만 보여준다.

### 지원 문항 유형
- 객관식 단일
- 객관식 중복
- 객관식 순위
- 객관식 척도
- 주관식 숫자
- 주관식 시간
- 주관식 비율 배분
- 주관식 문자

### 비지원 문항 유형
- 위 지원 목록에 포함되지 않는 유형
- 비지원·집계 불가 시 시각화 영역에 가운데 정렬된 안내 블록(`neutral-50` 배경)으로 `아직 지원하지 않는 문항 형식`을 표시한다. 글꼴은 **1-1** `비지원 안내 문구` 토큰을 따른다.

### 1-2. 컬러 팔레트 용도

#### 단일 색상 그래프
- 단일 색상을 사용하는 경우(가로/세로 막대 기본 차트, 가로/세로 롤리팝 기본 차트, 히스토그램 등) `neutral-700`을 기본으로 한다.

#### 여러 색상이 필요한 그래프
- **그룹 비교**: 그룹 순서대로 `color-1` → `color-2` → … → `color-20` 순환 할당. 최대 20개 그룹까지 색상 구분 가능.
- **객관식 단일 — 누적 막대 / 원형 그래프**: 보기 순서대로 `color-1` → `color-2` → … 순환 할당해 보기별 색상을 구분.

#### 순위 계열
- 누적 막대 차트에서 1/2/3순위 구간을 색상으로 구분할 때 사용한다. neutral 계열만 사용하되 명도로 순위를 구분한다.

| 순위 | 토큰 |
|------|------|
| 1순위 | `neutral-700` |
| 2순위 | `neutral-400` |
| 3순위 | `neutral-300` |
| 4순위 | `neutral-200` |
| 4순위 | `neutral-100` |

#### 척도 계열
- 1점(최저)은 `color-2`(붉은 계열), 중간점은 `neutral-300`, 최고점은 `color-1`(푸른 계열)을 기준으로 하고, 그 사이는 자연스러운 그라데이션 값으로 채운다.
- 5점 척도와 7점 척도는 아래 가이드를 따르고, 그 외 척도는 같은 방식으로 적당히 지정한다.

**5점 척도**

| 점수 | 토큰 |
|------|------|
| 1 | `color-2` |
| 2 | `low-3` |
| 3 | `neutral-300` |
| 4 | `high-3` |
| 5 | `color-1` |

**7점 척도**

| 점수 | 토큰 |
|------|------|
| 1 | `color-2` |
| 2 | `low-4` |
| 3 | `low-2` |
| 4 | `neutral-300` |
| 5 | `high-2` |
| 6 | `high-4` |
| 7 | `color-1` |

**그 외 척도**: 위 기준점(`color-2` → `neutral-300` → `color-1`)을 유지하면서 단계 수에 맞게 중간 토큰을 적절히 배정한다.

#### 박스플롯 계열
- 박스 내부 구간(Q1–Q2, Q2–Q3)은 두 구간을 같은 단일 색상으로 채운다.
- 단일 보기(응답자 전체)는 `neutral-400`을 사용한다.
- 그룹 비교는 그룹 비교 팔레트(`color-1` → `color-2` → … 순환)를 그대로 따른다.

#### 척도 평균 그래프 계열
- 분포 막대 배경의 각 구간 색상은 척도 분포 그래프와 동일한 색상을 사용하되, opacity 0.3을 적용한다.

### 2-3. 차트

#### 2-3-0. 공통

> **방향 정의**: 막대가 가로로 뻗는 차트를 **가로 막대**, 막대가 세로로 뻗는 차트를 **세로 막대**라 한다. 가로 막대에서 보기명은 왼쪽, 비율은 오른쪽에에 표시된된다. 세로 막대에서 보기명은 하단, 비율은 위에 표시된다.

- 막대 border-radius: `--radius-4` (4px). 누적 막대는 첫 번째·마지막 구간에만 양끝 적용.
- 수치 레이블: `Label-1-strong` (→ 1-1 타이포그래피 참조). 차트 요소가 너무 좁을 때는 표시하지 않음.
- 보기명: `Label-1` (→ 1-1 타이포그래피 참조), `neutral-900`
- 막대 그래프 (1~5번, 11번 히스토그램) 공통 축 스타일: 축 색상 `neutral-300` / 눈금 색상 `neutral-200`
- 막대 hover: opacity 0.85
- 막대 전환 애니메이션: `width 0.2s ease` (가로 막대) / `height 0.2s ease` (세로 막대)

#### 2-3-1. 가로 막대

보기(항목)별 **선택 비율·빈도**를 막대 길이로 나란히 비교할 때 쓴다.

**주요 요소**

| 항목 | 값 |
|------|----|
| 막대 트랙 너비 | 차트 영역 100% 기준 (막대 길이는 응답 비율에 비례) |
| 막대 높이 | 32px |
| 행 간격 | 8px |
| 보기명 위치 | 좌측 고정 열, text-right |
| 보기명 열 너비 | 차트 너비의 20%, 최소 120px, 최대 220px |

#### 2-3-12. 그룹별 비교

기준 문항으로 응답을 나눈 뒤, 그룹마다 값이 어떻게 다른지 한 화면에서 비교할 때 쓴다. 같은 차트 안에 **겹쳐** 보여 주는 방식과, **행을 세로로 쌓아** 나열하는 방식이 있다. 아래에 나온 차트만 그룹 비교를 지원한다.

**지원 차트**

| 차트 | 방식 |
|------|------|
| 가로 막대 | 겹침(도트) / 그룹 2개일 때 이중 막대 |
| 롤리팝(가로) | 겹침(도트) / 그룹 2개일 때 이중 롤리팝 |
| 100% 기준 누적 가로 막대 | 세로 나열(lane-group) |
| 척도 평균 그래프 | 세로 나열(lane-group) |
| 박스 플롯 | 세로 나열(lane-group) |

---

**(1) 가로 막대·롤리팝(가로) — 도트로 겹쳐 보기**

응답자 전체 기준 막대(또는 롤리팝)를 연한 회색(`neutral-200`)으로 깔아 두고, 그 위에 그룹별 값을 **지름 16px** 도트로 겹친다. 도트 색은 범례의 그룹 색과 같고, 테두리는 흰색이다. 도트를 누르면 그 그룹만 남기고 나머지는 흐리게 하며, 해당 그룹의 수치 레이블을 도트에 붙인다. 수치는 도트 **오른쪽**이 기본이나, 막대가 길어 오른쪽 여백이 부족하면 잘리지 않도록 도트 **왼쪽**에 둔다.

막대·축·수직 눈금선 등 겹침 모드의 골격은 일반 `#### 2-3-1. 가로 막대` 또는 `#### 2-3-6. 롤리팝 (가로)`와 같다.

---

**(2) 그룹이 둘일 때 — 이중 막대·이중 롤리팝**

범례에서 표시 중인 그룹이 **두 개만** 남으면, 범례 패널에 `두 그룹만 비교하기` 버튼이 노출된다. 이걸 쓰면 배경용 회색 차트는 숨기고, **보기(행)마다** 막대 또는 롤리팝 트랙을 **세로로 두 줄** 쌓아 두 그룹을 나란히 본다.

- 한 보기 안에서 두 막대(또는 두 롤리팝 트랙) 사이: **4px**
- 보기와 보기(행) 사이: **16px**
- 막대 쪽 축·눈금·수치 레이블 규칙은 `#### 2-3-1. 가로 막대`와 동일하다.
- 막대 채움색과 롤리팝 줄기·도트(헤드) 색은 모두 범례의 그룹 색과 같다.

---

**(3) 100% 누적·척도 평균·박스 — lane-group으로 세로 나열**

응답자 전체와 각 그룹을 **위에서 아래로** 다른 행에 그린다. 공통 레이아웃 클래스는 `.lane-group-chart`이며, 다음을 공유한다.

- **순서**: 맨 위는 항상 `응답자 전체` 한 행 → 가로 구분선(`lane-group-divider`, 좌측은 라벨 열 폭만큼 비움) → 그 아래로 그룹별 행.
- **행 사이**: 컨테이너 세로 `gap` **16px**. 구분선에는 위·아래 **8px** 마진을 더 둔다.
- **세로 치수**: **40px**. 누적 막대 트랙(`stack100-track`), 척도 분포/평균 전용 행, 박스 행, 평균 원 지름이 이 토큰에 맞춰진다.

**100% 기준 누적 가로 막대**  
막대 아래에 붙이던 보기명 레이블은 그룹 모드에서는 두지 않고, 보기 이름은 범례에서만 본다. 구간 색·비율 표기 등 나머지 골격은 `#### 2-3-3. 가로 막대 100% 기준 누적형`을 따른다.

**척도 평균 그래프**  
분포 막대 모드와 평균만 보는 모드 모두 위 lane-group 규칙을 쓴다. 그룹 나열에서는 평균 마커 위의 `평균` 텍스트와 원 안 숫자 레이블을 빼 간단히 한다.

**박스 플롯**  
박스 행 높이와 평균 원은 위와 같이 40px 토큰으로 맞춘다. 평균 쪽 레이블은 척도와 같이 줄이고, `Q1`·`Q2`·`Q3` 문구는 표시하지 않는다.

---

**그룹 비교를 쓰지 않는 차트**

세로 막대, 가로·세로 막대 누적형, 롤리팝(세로), 원형(파이), 히스토그램은 그룹별 보기를 제공하지 않는다.


### 2-4. 범례 패널

#### 2-4-1. 기본 기능

차트 우측 열에 배치되는 범례 패널이다.  

##### 2-4-1-1. 범례 레이아웃

`.result-visual`은 두 열 그리드(`grid-template-columns: minmax(0, 1fr) var(--viz-legend-width)`)로 차트와 범례를 나란히 두며, 차트 열이 남은 가로 공간을 차지하고 범례 열 너비는 `--viz-legend-width` 토큰에 따른다(위 **`2-1. 결과 카드` > 시각화 영역** 표의 범례 패널 너비와 동일).

범례의 기본 구성은 `체크박스 + 색상 swatch + 항목명`이다.  
체크박스는 단순히 현재 색상과 항목을 보여주는 장식이 아니라, 해당 항목을 차트와 표에서 표시할지 숨길지를 사용자가 직접 제어하는 토글이다.  
경우에 따라 체크박스나 일괄 액션을 두지 않는 범례도 있다(아래 **범례의 종류** 및 **`## 4. 문항 응답 형식별 시각화 규칙`** 참고).

체크박스는 `design_system.md`의 Checkbox 컴포넌트 스펙을 따른다.

##### 2-4-1-2. 범례의 유형

범례 **유형**마다 색을 대응시키는 대상·체크박스 유무·액션 버튼을 아래에 정리한다.
어떤 문항·차트에서 어떤 범례가 붙는지는 **`## 4. 문항 응답 형식별 시각화 규칙`**을 참고한다. 

| 범례 종류 | 색이 붙는 대상 | 체크박스 | 액션 버튼 |
|-----------|----------------|----------|-----------|
| **그룹 범례** | 비교 집단(그룹)마다 하나의 swatch | 있음 — 그룹 행마다, 차트·표에 그 그룹을 넣을지 고름 | `전체 선택`, `전체 해제`, `그룹 편집`, `두 그룹만 비교하기`※ |
| **순위 범례** | 1순위·2순위 등 순위마다 하나의 swatch | 있음 — 순위마다, 그 순위 응답을 차트·표에 넣을지 고름 | `전체 선택`, `전체 해제` |
| **보기 범례** | 응답 보기(선택지)마다 하나의 swatch | 없음 | 없음 |
| **점수 범례** | 척도 점수(및 코드북에 연결된 그 점수의 보기 문구)마다 하나의 swatch | 없음 | 없음 |

**보기 범례**와 **점수 범례**는 표시 전용으로, 클릭·호버 등 인터랙션이 없다.

- ※ **`두 그룹만 비교하기`**: **가로 막대**·**가로 롤리팝(순위)** 그룹 비교 모드, **척도형 여러 문항 비교**의 그룹 비교 모드에서, 범례에 남겨 둔(체크된) 그룹이 **정확히 둘**일 때만 버튼이 노출된다. 100% 기준 누적 가로 막대·척도 평균 그래프·박스 플롯 등 lane-group 방식 그룹 비교에는 두지 않는다(`#### 2-3-12. 그룹별 비교` **(2)** 참고).

범례 패널은 공통적으로, 필요하면 서로 다른 종류의 범례를 둘 이상 한 패널에 둘 수 있으며, 그럴 경우 구획 사이에 구분선을 둔다.

**범례 기본 스펙**

| 항목 | 기준 |
|------|------|
| 너비 | `var(--viz-legend-width)` (수치·식은 `2-1` 시각화 영역 표와 동일) |
| 패딩 | 없음 |
| **줄 사이** 간격 | 세로로 쌓인 **범례 행**과 행 사이, `gap: 8px` |
| **한 줄 안** 간격 | **같은 행**에서 체크박스·swatch·텍스트 사이, `gap: 8px` |
| 체크박스·swatch 크기 | 14px × 14px |
| swatch border-radius | 4px |
| 범례 항목 텍스트 | `Label-1`, `neutral-800` |
| 범례 항목 추가 텍스트 | `Label-1`, `neutral-600` (예: 점수 옆 보기 문구) |
| 비활성 텍스트 | `neutral-300` |
| 비활성 swatch | opacity 0.28 |
| 항목 블록 ↔ 액션 첫 줄 | 세로 간격 **24px** |
| 액션 버튼끼리 | 가로 간격 **8px** (줄 바꿈 가능) |
| `두 그룹만 비교하기` ↔ 위 액션 줄 | 세로 간격 **8px** |

##### 2-4-1-3. 범례 액션 버튼

**그룹 범례**·**순위 범례**에만 아래 버튼 묶음이 올 수 있다. **보기 범례**·**점수 범례**에는 액션 버튼을 두지 않는다(위 **범례의 종류** 표).

| 버튼 | 붙는 범례 유형 | 역할 |
|------|----------------|------|
| `전체 선택` | 그룹 범례, 순위 범례 | 그 범례 구획의 체크박스를 모두 켠다 |
| `전체 해제` | 그룹 범례, 순위 범례 | 모두 끈다 |
| `그룹 편집` | 그룹 범례만 | `범례 그룹 편집` 모달을 연다 |
| `두 그룹만 비교하기` | 일부 그룹 범례만 | 위 **범례의 종류** ※. 켜면: **가로 막대·가로 롤리팝(순위)**에서는 `#### 2-3-12. 그룹별 비교` **(2) 이중 막대·이중 롤리팝** 모드로 전환. **척도형 여러 문항 비교**에서는 각 문항 행에 그룹별 롤리팝 트랙을 세로로 쌓는 이중 롤리팝 형태로 전환. |

- `design_system.md`: `전체 선택`·`전체 해제`는 **`Button-2 Default`**, `그룹 편집`은 **`Button-2 Gray`**, `두 그룹만 비교하기`는 **`Button-2 Black`**.

#### 2-4-2. 범례 그룹 편집 모달

##### 2-4-2-1. 기본 구조

- 범례에서 `그룹 편집` 버튼을 누르면 `범례 그룹 편집` 모달이 열린다.
- 사용자는 여러 범례 항목을 하나의 그룹으로 묶어 차트와 데이터 테이블에서 비교 단위를 재구성할 수 있다.
- 모달은 **헤더**·**본문**·**푸터**로 나뉜다.

**헤더**

| 항목 | 기준 |
|------|------|
| 타이틀 | `범례 그룹 편집` |
| 닫기 버튼 | 헤더 우측 `×` 아이콘. 누르면 편집 내용을 저장하지 않고 모달을 닫는다. |

**본문**

스크롤 가능한 **단일 세로 리스트**다. 분석 기준의 그룹 값을 **원본 범례 순서**대로 나열하며, 행 종류는 아래 두 가지뿐이다.

| 행 종류 | 언제 쓰이는지 | 구성 |
|---------|----------------|------|
| **미배정 항목 행** | 아직 사용자 정의 그룹에 넣지 않은 값 | 체크박스 + swatch + 항목명(위 **`2-4. 범례 패널` > 범례 기본 스펙**; 체크박스는 `design_system.md` Checkbox). 드래그하여 기존 그룹으로 옮길 수 있음 |
| **그룹 아코디언** | 하나 이상의 범례 항목을 묶어 만든 사용자 정의 그룹 | 헤더 + 본문 |

**푸터**

푸터는 액션 버튼으로 이루어진다.

| 항목 | 기준 |
|------|------|
| 위치 | 모달 하단 |
| 구성 | 좌: `+` 아이콘 + `그룹 만들기` / 우: `전체 해제`, `편집 완료` |
| 버튼 스타일 | `그룹 만들기`·`전체 해제`는 `Button-2 Default`(비활성 시 `Button-2 Inactive`). `편집 완료`는 `Button-2 Black`(비활성 시 배경만 `neutral-300`으로 연해지며, Inactive variant는 쓰지 않음) |
| `그룹 만들기` | 본문에서 선택한 미배정 항목으로 새 아코디언 그룹을 만든다. 선택된 미배정 항목이 없으면 비활성 |
| `전체 해제` | 만든 그룹을 모두 없애 모든 항목을 미배정 행으로 되돌린다. 그룹이 하나도 없으면 비활성 |
| `편집 완료` | 현재 편집 내용을 차트·표에 적용하고 모달을 닫는다. 변경이 없으면 비활성 |

아래 **편집 흐름 — …** 절은 모달 안에서 사용자가 거치는 순서를 설명한다. 

##### 2-4-2-2. 편집 흐름 — 최초 진입

- 저장해 둔 사용자 정의 그룹이 없을 때, 모든 범례 항목이 **미배정 항목 행**만 나열된다. 그룹 아코디언은 없다.
- 사용자는 미배정 항목을 체크박스로 고를 수 있다.

##### 2-4-2-2. 편집 흐름 — 그룹 만들기

1. 미배정 행에서 묶을 항목을 체크박스로 **하나 이상** 선택한다.
2. `그룹 만들기` 버튼을 누른다.
3. 선택 항목이 **아코디언 그룹**으로 바뀐다.
   - 리스트에서는 멤버 중 **원본 순서가 가장 앞선 항목** 자리에 블록이 들어간다.
   - 아코디언은 **펼침** 상태로 표시된다.
   - 헤더 구성: 접기·펼치기, 그룹 swatch, 그룹명(한 줄 입력란), **이름 수정**·**그룹 해제** 아이콘.
   - 본문 구성: 멤버별 swatch·항목명, **항목 제거** 아이콘.
   - 그룹명 입력란에 입력 커서가 이동하고 텍스트는 전체 선택 상태다.
   - 그룹 swatch 색은 `customGroupDefs` 배열의 생성 순서에 따라 사용자 정의 그룹 전용 팔레트(`var(--color-11)` ~ `var(--color-18)`, 순환)에서 자동 배정된다.
4. 사용자가 그룹명을 입력·수정한다. `Enter` 또는 입력란 바깥 클릭으로 확정하며, 비워 두고 확정하면 `그룹1`, `그룹2`, … 순으로 기본 이름이 부여된다.
5. 선택 체크는 모두 해제된다. 아직 묶지 않은 값은 **미배정 항목 행**으로 남는다. 1~5를 반복해 그룹을 여러 개 만들 수 있다.

##### 2-4-2-2. 편집 흐름 — 그룹 수정/편집

그룹을 하나라도 만들면, 본문 리스트에는 **미배정 항목 행**과 **그룹 아코디언**이 원본 범례 순서대로 섞여 보인다. 그룹에 속한 값은 미배정 행으로 다시 나오지 않고, 해당 아코디언 **본문**에만 보인다. 이 상태에서 아래처럼 고칠 수 있으며, 변경이 생기면 `편집 완료`가 활성화된다.

- **그룹 이름 바꾸기** — 확정된 그룹명은 헤더에 **텍스트**로 보인다. **이름 수정** 아이콘을 누르면 한 줄 입력란으로 바뀌고, 이름을 고친 뒤 `Enter` 또는 입력란 바깥 클릭으로 다시 확정한다. 이름을 작성하지 않고 확정하면 `그룹1`, `그룹2`, … 순으로 기본 이름을 부여한다. 서로 다른 그룹이 같은 이름을 써도 된다.
- **기존 그룹에 항목 추가** — `그룹 만들기`와 별도로, 미배정 행을 드래그해 기존 아코디언 위에 놓으면 그 그룹에 추가할 수 있다. 드래그하여 놓으면 해당 그룹은 펼쳐진다.
- **기존 그룹의 항목 제거** — 아코디언 본문의 **항목 제거** 아이콘을 누르면 그 항목만 그룹에서 빠져 **미배정 항목 행**으로 돌아간다. 그룹에 남은 멤버가 없으면 그룹은 자동으로 해제된다.
- **그룹 삭제** — 아코디언 헤더의 **그룹 해제** 아이콘을 누르면 해당 그룹은 삭제되고, 속했던 항목은 모두 **미배정 항목 행**으로 돌아간다.
- **모든 그룹 삭제** — 푸터의 **전체 해제** 버튼을 누르면 만든 그룹이 모두 없어지고, 모든 항목이 미배정 행으로 돌아간다.

##### 2-4-2-2. 편집 흐름 — 편집 완료 후 적용

- 푸터의 `편집 완료` 버튼을 누르면 편집된 그룹 구성이 적용되고 모달이 닫힌다. 모달을 연 뒤 변경을 하지 않았으면 버튼은 활성화되지 않는다.
- 적용된 그룹은 차트·데이터 테이블 양쪽에서 **새 범례 항목** 하나로 취급한다. 그룹에 넣은 원본 범례 항목은 차트·표 집계 단위로는 더 이상 각각 따로 두지 않는다. 그룹에 넣지 않은 항목은 원래 범례 항목으로 그대로 남는다.

**그룹 편집 후 대시보드 상의 범례 표시**

- **사용자 정의 그룹**은 범례 패널에서 **그룹 한 줄**로 보인다. 체크박스 + 그룹 색 swatch + 사용자가 지정한 그룹명(예: `그룹1`, `그룹2`)이며, 위 **`2-4. 범례 패널` > 범례 기본 스펙**을 따른다.
- 그룹에 묶었던 **원본 범례 항목**은 같은 그룹 줄 **아래에 들여쓴 하위 목록**으로 함께 표시한다. 각 항목은 원래 범례와 같이 swatch + 항목명(예: `세대: Z세대(1996-2001)`)만 보이고, **체크박스는 없다**. 표시·숨김은 그룹 줄의 체크박스로만 제어한다.
- 하위 항목 swatch는 원본 그룹 색을 유지하되 `opacity: 0.7`을 적용해 그룹 줄 swatch보다 연하게 구분한다. 하위 항목 텍스트는 `neutral-700`이다. 그룹을 끄면(비활성) 하위 목록 전체에 `opacity: 0.28`을 적용한다.
- 그룹에 넣지 않은 원본 범례는 기존처럼 **그룹 범례 한 줄**(체크박스 + swatch + 항목명)로만 나열된다.

##### 2-4-2-2. 편집 흐름 — 재진입

- 이전에 `편집 완료`로 저장해 둔 사용자 정의 그룹이 있는 상태에서 모달을 열면, **그룹 수정/편집**과 같은 혼합 리스트 상태로 시작한다. **최초 진입**(미배정만 있는 상태)으로 돌아가지 않는다.

##### 2-4-2-2. 편집 흐름 — 예외 처리

- 선택 항목이 없으면 그룹 생성은 불가하다.
- 동일 항목을 둘 이상의 그룹에 중복 포함할 수 없다.
- 그룹명 중복은 허용한다.
- 빈 그룹은 허용하지 않으며, 항목이 모두 제거된 경우 그룹 해제로 처리한다.

### 2-5. 데이터 테이블

문항·차트 유형별 **열 구성·행 구조·합계 행·하단 안내 문구**는 **`## 4. 문항 응답 형식별 시각화 규칙`** 각 절의 `#### 데이터 테이블`을 따른다. 이 절은 모든 표에 공통으로 적용되는 **레이아웃·타이포·보더·토글·복사** 스타일만 정의한다.

#### 2-5-1. 배치

| 항목 | 기준 |
|------|------|
| 위치 | 차트 행 아래(→ `#### 시각화 영역`의 차트↔테이블 간격 48px) |
| 섹션 좌·우 패딩 | 16px |

#### 2-5-2. 테이블 공통

| 항목 | 기준 |
|------|------|
| 너비 | 100%, `border-collapse: collapse` |
| 셀 패딩 | 8px 16px |
| 기본 정렬 | 좌측 (`text-align: left`) |
| 줄바꿈 | `white-space: nowrap` |
| 글꼴(본문·일반 셀) | `Label-2`, `neutral-700`, line-height 1.6 |
| 글꼴(헤더·합계) | `Label-2-strong`, `neutral-700` |
| 행 구분선 | 모든 `th`·`td` 하단 1px solid `neutral-300` |
| 본문 마지막 행 | 하단 보더 없음 (`tbody` 마지막 `tr`의 `td`) |

수치 열(`응답 비율`, `응답 수`, 평균 등)에는 우측 정렬과 고정폭 숫자(`tabular-nums`)를 적용한다.

#### 2-5-3. 첫 번째 열·그룹 구분

| 항목 | 기준 |
|------|------|
| 첫 번째 열(보기·문항명 등) | 가로 스크롤 시 좌측 고정(sticky), 배경 `neutral-50` |
| 첫 번째 열 우측 구분선 | 열 오른쪽에 1px solid `neutral-300` |
| 그룹·순위 블록 경계 | `group-col` 열 왼쪽 1px solid `neutral-300` |

다중 헤더 행이 있을 때, 첫 번째 열을 제외한 일부 헤더·셀은 인접 보더가 겹치지 않도록 왼쪽 보더를 생략한다.

#### 2-5-4. 헤더·합계 행

| 구분 | 기준 |
|------|------|
| 헤더(`th`) | 배경 `neutral-50`, `Label-2-strong` |
| 합계 행 | 배경 `neutral-50`, `Label-2-strong`, 상단 1px solid `neutral-300` (첫 번째 열도 동일 배경). 표의 마지막 행이므로 하단 보더는 표시하지 않음 |

#### 2-5-5. 평균값 셀

척도·파생 척도 등에서 평균 수치 셀에 적용한다. 비율 배분 표는 예외로 일반 수치 셀과 동일한 굵기·색을 유지한다.

| 대상 | 글꼴 | 텍스트 색상 |
|------|------|-------------|
| 일반(`.mean-value`) | `Label-2-strong` | `neutral-900` |
| 비율 배분 표의 평균 셀 | `Label-2` | `neutral-700` |

여러 문항 척도 비교 표에서 평균 열(`mean-col`) 왼쪽에는 `group-col`과 같이 1px solid `neutral-300`을 둔다.

#### 2-5-6. 보충 설명

| 항목 | 기준 |
|------|------|
| 글꼴 | `Caption-1`, `neutral-600` |
| 위치 | 테이블 래퍼 바로 아래 |
| 테이블과의 간격 | 8px |

문구 내용은 문항 형식별로 `## 4`에 정의한다(예: 객관식 중복 비율 합계 안내).

#### 2-5-7. 데이터 테이블 영역 헤더(펼치기/숨기기·복사)

테이블 위에 **데이터 테이블 펼치기/숨기기** 토글 버튼과 **복사** 아이콘 버튼을 한 줄로 둔다(`justify-content: space-between`).

**데이터 테이블 펼치기/숨기기**

- 아이콘(위·아래 화살표)을 문구 왼쪽에 두고, 아이콘과 문구 사이 간격은 4px.
- 글꼴: `Label-1-strong`, `neutral-700`.
- **펼침**(테이블 표시 중): 위 화살표 + `데이터 테이블 숨기기`
- **숨김**(테이블 접힘): 아래 화살표 + `데이터 테이블 펼치기`
- 기본 상태는 **펼침**이다. 접으면 테이블 본문(래퍼·보충 설명 포함)만 숨긴다.

**데이터 테이블 복사**

- 아이콘 16×16px.
- 누르면 **같은 카드에 표시 중인 데이터 테이블**을 클립보드에 넣는다. 붙여넣기 시 표 형식(HTML)과 탭으로 구분된 텍스트(TSV)를 함께 제공하며, 환경에 따라 텍스트만 복사될 수 있다.
- 복사 대상에는 **보기·비율·평균 등 표에 보이는 값**이 포함되고, **`응답 수(명)` 열은 제외**한다. 펼치기/숨기기·보충 설명 문구는 포함하지 않는다.

## 3. 데이터 표시 규칙

### 비율과 반올림
- 비율은 내부 계산값을 유지하고, 표시 시점에만 소수점 첫째 자리까지 반올림한다.
- 반올림은 부동소수점 오차를 최소화하기 위해 `Math.round((value + 1e-10) * 10) / 10` 방식으로 처리한다.
- 응답자 수 `N`은 반올림하지 않고 정수 그대로 표시한다.
- 같은 문항 안에서는 차트, 툴팁, 표가 동일한 비율 표시 규칙을 사용한다.
- 툴팁에서는 응답 비율을 `xx.x%`, 응답 수를 `N=xxx` 형식으로 표시한다.

### 보기 정렬
- 기본 보기 순서는 코드북의 보기 정의 순서를 따른다.
- 코드북에 없는 예외 응답이 있으면 기존 보기 뒤에 추가한다.
- 전체 기준으로 응답 0건인 보기는 기본 차트와 표에서 숨긴다.
- 그룹 비교에서도 전체 기준 0건 보기는 함께 숨긴다.

### 그룹핑 규칙
- 그룹 비교 시 그룹명은 `분석 기준 문항명: 그룹 값` 형식으로 표기한다.
  - 예: `성별: 여`, `세대: Z세대`
- 그룹별로 차트를 세로로 나열하는 경우, 좌측에 표시되는 그룹명 레이블은 가로 막대 보기명과 동일한 레이아웃을 사용한다. 즉, 좌측 고정 열에 `text-right`로 배치하고 너비는 차트 너비의 20%, 최소 120px, 최대 220px으로 한다.
- 그룹 비교가 가능한 화면에서는 체크박스로 그룹별 표시 여부를 제어한다.
- 개별 그룹 on/off가 가능하다.
- 범례에 붙는 액션 버튼(`전체 선택`, `전체 해제`, `그룹 편집`, `두 그룹만 비교하기` 등)의 **유형·조합**은 **`2-4. 범례 패널` > 범례의 종류·범례 액션 버튼**을 본다.
- 해제된 그룹은 범례에서 색과 텍스트를 흐리게 표시해 비활성 상태로 구분하고, 차트와 표에서는 숨긴다.

### 기타 응답 보기
- 기타 응답용 텍스트 컬럼이 있으면 데이터 테이블의 해당 보기명 옆에 `응답 보기` 버튼을 표시할 수 있다.
- 버튼을 누르면 기타 자유응답 목록을 모달로 보여준다.
- 모달에는 중복 제거 없이 실제 수집된 텍스트 목록을 그대로 보여준다.

### 차트 유형 옵션 요약

| 주관식 문자 | 응답 목록 박스 | — | 키워드 검색 |

### 4-1. 객관식 단일

#### 집계 기준
- 각 보기별 선택 응답 수를 집계하고, 전체 응답 수 대비 비율을 계산한다.

#### 컨트롤바
- 그래프 유형 드롭다운 + 응답 비율 내림차순 정렬 옵션(체크박스 + 텍스트). 기본은 코드북 순서다.

#### 차트
- 기본 차트는 가로 막대. 드롭다운으로 세로 막대 / 100% 기준 누적 가로 막대 / 원형(파이)으로 전환 가능.
- 객관식 단일의 가로 막대는 막대 트랙 배경색으로 `neutral-50`을 사용한다.
- `100% 기준 누적 가로 막대` 선택 시 막대·구간의 **치수·배치 골격**은 `#### 2-3-3. 가로 막대 100% 기준 누적형`을 따른다. 보기별 구간 색·하단 보기명·구간 내부 수치의 **상세**는 아래 표를 따른다.

| 항목 | 값 |
|------|----|
| 구간 색상 | 보기 순서대로 `color-1` → `color-2` → … (→ 아래 **범례** 절과 동일) |
| 구간 하단 보기명 | 각 구간 하단, 구간 너비에 맞춰 표기, 세그먼트가 지나치게 좁으면 미표시 |
| 보기명 좌우 패딩 | 8px |
| 구간 내부 수치 레이블 색상 | `white` |

#### 툴팁
- 기본: `보기명`, `비율`, `응답 수`
- 그룹 비교: `그룹명`, `보기명`, `비율`, `응답 수`

#### 범례
- 가로 막대 (기본): 범례 없음 (단색 막대)
- 세로 막대 / 100% 기준 누적 가로 막대: 보기별 색상 범례 표시 (보기 순서대로 `color-1` → `color-2` → … 순환)
- 원형(파이) 차트: 범례 섹션에 색상 swatch + 보기명만 표시한다 (체크박스 없음)

#### 그룹 비교
- 가로 막대 / 100% 기준 누적 가로 막대에서만 지원한다.
- `#### 2-3-12. 그룹별 비교` 섹션 참조.

#### 데이터 테이블
- 전체: `보기`, `응답 비율(%)`, `응답 수(명)`
- 그룹 비교: 전체 + 각 그룹의 `응답 비율(%)`, `응답 수(명)`
- 기타 응답용 텍스트 컬럼이 있으면 데이터 테이블의 `기타` 행 옆에 `응답 보기` 버튼을 표시한다. 버튼을 누르면 기타 자유응답 텍스트 목록을 모달로 보여준다.

### 4-2. 객관식 중복

#### 집계 기준
- 보기별 expanded 이진 컬럼을 기준으로 각 보기의 선택 비율과 응답 수를 독립 집계한다.
- 선택값으로 `1`, `y`, `yes`, `true`, `선택`을 모두 선택 처리한다.

#### 컨트롤바
- 그래프 유형 드롭다운 + 응답 비율 내림차순 정렬 옵션(체크박스 + 텍스트). 기본은 코드북 순서다.

#### 차트
- 기본 차트는 가로 막대. 드롭다운으로 세로 막대로 전환 가능.
- 객관식 중복의 가로 막대는 막대 트랙 배경색으로 `neutral-50`을 사용한다.

#### 툴팁
- 기본: `보기명`, `비율`, `응답 수`
- 그룹 비교: `그룹명`, `보기명`, `비율`, `응답 수`

#### 범례
- 기본: 없음 (단색 막대)
- 그룹 비교 시: 그룹별 범례 표시

#### 그룹 비교
- 가로 막대에서만 지원한다.
- `#### 2-3-12. 그룹별 비교` 섹션 참조.

#### 데이터 테이블
- 전체: `보기`, `응답 비율(%)`, `응답 수(명)`
- 그룹 비교: 전체 + 각 그룹의 `응답 비율(%)`, `응답 수(명)`
- 하단 안내 문구 : `객관식 중복 응답 문항으로, 보기별 비율 합계는 100%를 초과할 수 있습니다.`

### 4-3. 객관식 순위

#### 집계 기준
- 순위 문항은 순위별 expanded 컬럼을 기준으로 집계한다.
- 순위 expanded 컬럼명은 `__1순위` 표준 형식을 우선 사용한다. legacy 호환을 위해 `_1순위` 형식도 인식한다.
- 응답자 수 `N`은 raw 값이 있거나 순위 expanded 값이 하나라도 있으면 포함한다.
- 순위별 비율 계산 모수는 해당 문항에 참여한 전체 응답자 수를 사용한다.

#### 가중 점수 계산
- 가중치는 최하위 순위부터 1점씩 시작해 순위가 올라갈수록 2씩 증가한다.
  - 예: 3순위까지인 경우 1순위=5점, 2순위=3점, 3순위=1점
  - 예: 4순위까지인 경우 1순위=7점, 2순위=5점, 3순위=3점, 4순위=1점
- 보기별 가중 평균 점수 = 각 순위 응답 수 × 해당 순위 가중치의 합 ÷ 전체 응답자 수
- 종합 순위는 가중 평균 점수 기준 내림차순. 동률이면 공동 순위로 표시한다.

#### 비순위 응답 처리
- raw에는 값이 있지만 순위별 expanded에 실제 순위값이 없는 응답은 비순위 응답으로 본다.
- 전체 응답 수에는 포함하되, 가중 평균 점수와 종합 순위 계산에는 포함하지 않는다.

#### 기타 보기 처리
- `기타` 보기는 응답 수와 표에는 포함하되, 종합 순위 계산과 차트 표시(visibleOptionOrder)에서는 제외한다.

#### 컨트롤바
- 가중 평균 점수 내림차순 정렬 옵션(체크박스 + 텍스트). 기본은 코드북 순서다.
- 가로/세로 방향 전환: 드롭다운 (컨트롤바 좌측). 롤리팝 ↔ 누적 막대 전환: 토글 버튼 (드롭다운 우측).

#### 차트
- 기본 차트는 가로 롤리팝.
- 가로축 범위: 0점(좌끝) ~ 가중 평균 점수의 **만점(최고 점수)**에 맞춘 우끝. 만점은 순위 수에 따라 결정된다. 예: 3순위까지면 5점, 4순위까지면 7점.
- 각 보기의 우측 레이블에는 `가중 평균 점수(종합 순위)` 표기. 예: `3.2(1위)`
- 누적 막대 차트에서는 각 보기 안에서 `1순위`, `2순위`, `3순위` 등을 색상 구간으로 나눠 표시한다.
- 차트 하단 안내 문구: 가중 평균 산출 방식을 문장으로 풀어쓰고 산출식과 만점을 병기한다. 예: `각 보기의 가중 평균 점수는 순위별 응답 수에 가중치를 곱해 합산한 뒤 전체 응답자 수로 나눈 값입니다. 예) 3순위까지 선택하는 경우: (1순위×5 + 2순위×3 + 3순위×1) ÷ 전체 응답자 수 (만점: 5점)`

#### 툴팁
- 롤리팝: `보기명`, `가중 평균 점수`, `종합 순위`
- 누적 막대 순위 구간: `보기명`, `해당 순위`, `비율`, `응답 수`

#### 범례
- 기본: 누적 막대 보기에서만 순위별 범례를 표시한다.

#### 그룹 비교
- 가로 롤리팝에서만 지원한다.
- `#### 2-3-12. 그룹별 비교` 섹션 참조.

#### 데이터 테이블
- 보기별 순위별 `응답 비율(%)`, `응답 수(명)` + 우측에 `가중 평균`, `종합 순위` 컬럼
- 그룹 비교: 전체 순위 정보 + 그룹별 순위 정보
- 합계 행의 `가중 평균`, `종합 순위`는 `-`로 표시한다.

### 4-4. 객관식 척도

척도 문항은 `raw 척도`와 `derived 척도`를 구분해 처리한다.

#### raw 척도 집계 기준
- 각 점수별 응답 수를 집계하고, 전체 응답 수 대비 비율과 평균값을 계산한다.

#### raw 척도 컨트롤바
- 차트 유형 드롭다운: `100% 기준 누적 막대` / `원형(파이)` 전환.
- 분포 보기 ↔ 평균 보기 전환: 토글 버튼. `100% 기준 누적 막대` 선택 시에만 활성화하며, 원형 선택 시 비활성화한다.
- 홀수 척도는 `중간값 제외 보기` 체크박스를 추가 제공한다. 원형 보기에서는 비활성화한다.
- `중간값 제외 보기`는 `5점 척도`에서는 `3점`, `7점 척도`에서는 `4점` 막대를 차트에서만 숨긴다.
- 이 기능은 시각적 표현에만 적용한다. 숨겨진 막대를 제외한 나머지 막대가 100%를 다시 채워 보이지만, 각 점수의 응답 비율 숫자 자체는 원본 값을 유지하며 바뀌지 않는다.
- 안내 문구 `중간값 제외 보기는 차트에서만 시각적으로 제외하며, 응답 비율의 계산 모수와 원본 수치는 바뀌지 않습니다.`를 컨트롤바 하단에 표시한다.

#### raw 척도 차트
- 분포 보기: 점수별 응답 분포를 가로 100% 누적 막대로 표시. 하위/상위 척도 응답 합계 요약 함께 표시.
- 평균 보기: 척도 개수만큼 눈금이 있는 축 위에 평균 마커 표시. (`100% 기준 누적 막대` 선택 시에만 전환 가능)
- 원형(파이) 보기: 점수별 응답 비율을 원형 차트로 표시. 색상은 척도 계열 컬러 적용. 범례에 체크박스 없이 색상 swatch + 점수 + 레이블 표시.
- 분포 보기(가로 100% 누적 막대·원형)의 구간 색상은 1-2 **척도 계열**을 따른다.

#### raw 척도 — 분포 보기(가로 100% 누적 막대) 레이블·색 상세

막대 높이·구간 너비·구간 내부 수치 중앙 배치·하단 레이블 ↔ 막대 간격·하단 이중 행(텍스트 레이블 `Label-1`, 수치 레이블 `Label-1-strong`, 줄 간격 0) 등 **공통 치수**는 `#### 2-3-3. 가로 막대 100% 기준 누적형`을 따른다. 아래는 척도 분포 보기에만 해당한다.

객관식 척도 분포 보기에서는 보기명 대신 하위/상위 척도 합계를 막대 양끝 하단에 표시한다.

| 항목 | 값 |
|------|----|
| 구간 색상 | 1-2 **척도 계열** |
| 구간 내부 수치 레이블 색상 | `white` |
| 하위 척도 합계 레이블 | 막대 좌끝 하단, 좌측 정렬 |
| 상위 척도 합계 레이블 | 막대 우끝 하단, 우측 정렬 |
| 레이블 구조 | 텍스트("하위" / "상위")와 수치(응답 비율)를 세로로 나열. 텍스트 ↔ 수치 간격 0 |
| 레이블 ↔ 막대 gap | 8px (`#### 2-3-3`과 동일) |
| 텍스트 스타일 | `Label-1` `neutral-700` |
| 수치 스타일 | `Label-1-strong` `neutral-700` |

#### raw 척도 툴팁
- 분포 보기: `점수값 또는 점수 레이블`, `비율`, `응답 수` (그룹 비교 시 `그룹명` 추가)
- 평균 보기: `평균값`, `응답 수` (그룹 비교 시 `그룹명` 추가)

#### raw 척도 범례
- 점수별 색상 / 점수 / 레이블을 표시한다. 예: `붉은색 swatch + 1점 + 전혀 그렇지 않다`

#### raw 척도 그룹 비교
- 그룹 비교 시에는 같은 형식의 차트를 그룹별로 세로로 나열해 비교한다.
- 그룹별 차트 간 세로 간격은 16px로 유지한다.
- 그룹 범례 및 비교 컨트롤은 범례 영역이 아니라 차트 영역 상단에 배치한다.
- 그룹 비교 컨트롤과 차트 사이 간격은 40px로 유지한다.

#### raw 척도 데이터 테이블
- 단일 전체: `점수`, `응답 비율(%)`, `응답 수(명)`, `평균` (평균 값은 합계 행에만 표시)
- 단일 그룹 비교: 전체 + 각 그룹의 `점수`, `응답 비율(%)`, `응답 수(명)`, `평균` (평균은 각 블록 합계 행에만 표시)

#### derived 척도 집계 기준
- 계산된 연속형 점수를 기준으로 최소값, 최대값, 평균, 사분위수(`Q1`, `Q2`, `Q3`)를 계산한다.

#### derived 척도 컨트롤바
- 분포 보기 ↔ 평균 보기 전환: 토글 버튼.

#### derived 척도 차트
- `type = scale` 이면서 `role = derived`인 경우 derived 척도로 본다.
- 분포 보기: 박스플롯. `Q1`, `Q2`, `Q3` 마커 하단에 레이블과 값을 세로로 표시한다. (→ **2-3-10. 박스플롯** 참조)
- 평균 보기: 척도 평균 그래프. (→ **2-3-9. 척도 평균 그래프** 참조)

#### derived 척도 툴팁
- 사분위수 마커: `사분위명`, `사분위값` (툴팁에서는 `Q1(하위 25%)`, `Q2(중앙값)`, `Q3(상위 25%)` 형식)
- 평균 보기: `평균값`, `응답 수` (그룹 비교 시 `그룹명` 추가)

#### derived 척도 범례
- 없음

#### derived 척도 그룹 비교
- 그룹 비교 시에는 같은 형식의 차트를 그룹별로 세로로 나열해 비교한다.
- 그룹별 차트 간 세로 간격은 16px로 유지한다.
- 그룹 범례 및 비교 컨트롤은 범례 영역이 아니라 차트 영역 상단에 배치한다.
- 그룹 비교 컨트롤과 차트 사이 간격은 40px로 유지한다.

#### derived 척도 데이터 테이블
- 단일: `문항명`, `최소값`, `Q1`, `Q2(중앙값)`, `Q3`, `최대값`, `평균`, `응답 수(명)`
- 단일 그룹 비교: `그룹명`, `최소값`, `Q1`, `Q2(중앙값)`, `Q3`, `최대값`, `평균`, `응답 수(명)` 구조로 표시하고, 표 상단에 `전체` 행을 별도로 둔다.

#### 척도형 여러 문항 비교
- 같은 척도 길이의 문항끼리 `여러 문항 한 번에 비교하기` 사용 가능.
- 여러 문항 비교 대상에는 `raw 척도`와 `derived 척도`가 함께 포함될 수 있다.
- 비교 대상 문항이 모두 같은 척도형이면 버튼 자동 활성화. 하나라도 다르면 비활성화.
- 그룹이 없으면 `분포 보기`와 `평균 보기`를 모두 사용할 수 있다.
- 그룹이 있으면 `분포 보기`는 사용할 수 없고 `평균 보기`로 고정한다.
- 여러 문항 비교 대상에 `raw 척도`가 하나라도 포함되어 있으면 `중간값 제외 보기` 체크박스를 활성화한다.
- 여러 문항 비교 (그룹 없음) 데이터 테이블:
- `raw 척도` 문항은 각 문항별 `문항명`, 각 점수별 `응답 비율(%)`, `응답 수(명)`, `평균`, `전체 응답 수(명)`을 표시한다.
- `derived 척도` 문항은 각 문항별 `문항명`, `평균`, `응답 수(명)`을 표시한다.
- 여러 문항 비교 (그룹 있음) 데이터 테이블: `raw 척도`와 `derived 척도` 모두 각 문항별 `문항명`, `전체 평균`, `전체 응답 수(명)`, 각 그룹의 `평균`, `응답 수(명)`을 표시한다.

### 4-5. 주관식 숫자

#### 집계 기준
- 숫자 응답값을 기준으로 최소값, 최대값, 평균, 사분위수(`Q1`, `Q2`, `Q3`)를 계산한다.
- 히스토그램 보기에서는 구간별 응답 수와 비율을 계산한다.

#### 컨트롤바
- 기본은 박스플롯이다.
- 박스플롯 ↔ 히스토그램 전환: 토글 버튼 (`요약 보기` / `분포 보기`). 그룹 비교 시 박스플롯으로 고정.
- 안내 문구 `구간 시작값과 간격을 조정해 분포를 원하는 기준으로 확인할 수 있습니다.`는 `분포 보기` 선택 상태일 때만 컨트롤바 하단에 표시한다.

#### 차트 — 박스플롯
- 최소값~최대값 whisker, `Q1~Q3` box, 중앙값 기준선, 평균 마커를 표시한다.
- 평균 마커는 중앙값 기준선과 겹칠 수 있으므로, 중앙값 기준선보다 위 레이어에 렌더링한다.
- 각 요소 의미를 설명하는 안내 문구를 함께 제공한다.

#### 차트 — 히스토그램
- 사용자가 구간 시작값과 구간 간격을 조정할 수 있다.
- 평균 마커를 표시한다.

#### 툴팁
- 히스토그램: `구간 범위`, `응답 수`
- 평균 마커: `평균값`, `응답 수`
- 사분위수 마커: `사분위명`, `사분위값` (툴팁에서는 `Q1(하위 25%)`, `Q2(중앙값)`, `Q3(상위 25%)` 형식)
- 박스플롯: `최소값`, `Q1(하위 25%)`, `Q2(중앙값)`, `Q3(상위 25%)`, `최대값`, `평균값`

#### 범례
- 없음

#### 그룹 비교
- 그룹 비교 시에는 박스플롯을 그룹별로 세로로 나열해 비교한다.
- 그룹별 차트 간 세로 간격은 16px로 유지한다.

#### 데이터 테이블
- 전체: `문항명`, `최소값`, `Q1`, `Q2(중앙값)`, `Q3`, `최대값`, `평균`, `응답 수(명)`
- 그룹 비교: `그룹명` 기준, 표 상단에 전체 행 별도 표시

### 4-6. 주관식 비율 배분

#### 집계 기준
- raw 컬럼에는 배분 항목 순서대로 `40|60`처럼 파이프 구분 숫자값이 들어갈 수 있다.
- expanded 컬럼이 있으면 `{원문항 라벨}__{배분 항목명}` 컬럼의 항목별 숫자값을 우선 사용한다.
- expanded 컬럼이 없으면 raw 값의 파이프 순서를 코드북의 `response_options` 순서에 맞춰 해석한다.
- 각 응답자의 항목별 합계가 100인 응답만 유효 응답으로 집계한다.
- 항목별 평균 배분값은 유효 응답자별 항목 값을 평균해 계산한다.

#### 차트
- 가로 100% 누적 막대. 각 구간 너비는 항목별 평균 배분값을 나타낸다.
- 막대·구간의 **치수·배치 골격**은 `#### 2-3-3. 가로 막대 100% 기준 누적형`을 따른다.
- 각 막대 하단에 항목별 레이블과 평균 배분값을 표시한다. 하단 레이블이 두 줄로 쌓이는 경우 **타이포그래피·정렬 계열**은 4-4 **raw 척도 — 분포 보기(가로 100% 누적 막대) 레이블·색 상세**에 정한 이중 행 규칙(텍스트 `Label-1`, 수치 `Label-1-strong`, 줄 간격 0 등)과 동일하게 적용한다.

#### 툴팁
- `그룹명`, `배분 항목`, `평균 배분값(%)`, `응답 수`

#### 범례
- 항목 구간은 배분 항목별 색상으로 구분한다. 색상은 `color-1`부터 항목 순서대로 적용한다.

#### 그룹 비교
- 그룹 비교 시에는 그룹별 가로 누적 막대를 세로로 나열한다.

#### 데이터 테이블
- 전체: `배분 항목`, `평균 배분값(%)`, `응답 수(명)`
- 그룹 비교: 전체 + 각 그룹의 `평균 배분값(%)`, `응답 수(명)`

### 4-7. 주관식 시간

#### 집계 기준
- 코드북에서 해당 문항의 `{question_label}_분환산` derived 행의 컬럼 값(분 단위 정수)을 집계에 사용한다.
- 집계 방식(최소값·최대값·평균·사분위수)은 주관식 숫자와 동일하다.

#### 차트
- 주관식 숫자와 동일하게 박스플롯(기본) / 히스토그램(분포 보기)을 사용한다.
- 모든 수치 레이블은 분 단위 값을 `hh:mm` 형식으로 변환해 표시한다. (예: 540분 → `09:00`)

#### 툴팁
- 주관식 숫자와 동일. 수치는 `hh:mm` 형식으로 표시한다.

#### 범례
- 없음

#### 그룹 비교
- 주관식 숫자와 동일하게 박스플롯을 그룹별로 세로로 나열한다.

#### 데이터 테이블
- 주관식 숫자와 동일한 구조. 수치는 `hh:mm` 형식으로 표시한다.
- 전체: `문항명`, `최소값`, `Q1`, `Q2(중앙값)`, `Q3`, `최대값`, `평균`, `응답 수(명)`
- 그룹 비교: `그룹명` 기준, 표 상단에 전체 행 별도 표시

### 4-8. 주관식 문자

#### 집계 기준
- 비어 있지 않은 텍스트 응답을 유효 응답으로 본다.
- 응답 텍스트는 중복 제거 없이 실제 수집된 값을 그대로 표시한다.

#### 컨트롤바
- 키워드 검색 input을 제공한다.
- 사용자가 키워드를 입력하면 해당 키워드를 포함하는 응답만 즉시 필터링한다.
- 검색어가 비어 있으면 전체 유효 응답을 표시한다.

#### 응답 목록
- 차트 대신 스크롤 가능한 응답 목록 박스를 표시한다.
- 박스 상단에는 `응답 {검색 결과 수}건 / 전체 {전체 유효 응답 수}명` 형식의 요약을 표시한다.
- 박스 본문에는 각 응답을 한 줄 또는 문단 단위로 세로 나열한다.
- 응답이 많으면 박스 내부에서만 세로 스크롤한다.

#### 툴팁
- 없음

#### 범례
- 없음

#### 그룹 비교
- 지원하지 않는다.

#### 데이터 테이블
- 별도 데이터 테이블은 표시하지 않는다.

## 5. 상호작용 규칙

### 그룹 on/off
- 그룹 비교 차트에서 표시할 그룹을 사용자가 직접 선택할 수 있다.
- 선택 해제된 그룹은 범례에서는 흐리게 표시되고, 차트와 표에서는 숨겨진다.
- 체크 상태 변경은 현재 표시 중인 차트와 데이터 테이블에 즉시 반영한다.

### 그래프 유형 전환
- 드롭다운을 사용하는 차트 유형에서는, 문항 형식별로 허용된 그래프 유형 옵션을 선택하면 즉시 재렌더링한다.
- 정렬 변경은 현재 선택된 그래프 유형에 동일하게 적용한다.

### 사용자 정의 그룹 편집
- 여기서의 `그룹 편집`은 `그룹 비교`의 그룹과 다른 개념이다. 버튼 이름은 `그룹 편집`이지만, 기능 의미는 기존 응답 보기(항목) 여러 개를 사용자가 원하는 `사용자 정의 묶음`으로 다시 묶는 것이다.
- 이 문서에서는 이 기능을 설명할 때 `사용자 정의 묶음`이라는 용어를 사용한다.
- 예: `직급` 문항의 보기인 `팀장`, `임원`을 하나의 사용자 정의 묶음 `리더`로 합쳐 볼 수 있다.
- 그룹 범례 액션 영역의 `그룹 편집` 버튼을 클릭하면 모달을 열어 사용자 정의 묶음의 생성/삭제/이름 변경/항목 매핑을 수행한다.
- 모달을 닫으면 변경 사항을 즉시 반영한다.
- 묶음 적용 상태에서는 범례에서 각 응답 보기 항목이 어느 사용자 정의 묶음에 속하는지(색 인덱스 포함) 확인할 수 있다.
- 묶음 해제는 `원래대로 보기` 버튼으로 즉시 복귀한다.

## 6. 기타 응답 처리 규칙

### 기타 텍스트 컬럼 탐색
- 기타 자유응답 텍스트는 전용 expanded 컬럼에서 읽는다.
- 현재 구현은 아래와 같은 이름 패턴을 순차적으로 허용한다.
  - `__기타_텍스트`
  - `__기타 텍스트`
  - `_기타_텍스트`
  - `_텍스트`
- 위 패턴에 정확히 맞지 않아도, 문항명과 기타 텍스트 의미가 함께 드러나는 경우 느슨한 fallback 탐색을 수행한다.

### 기타 응답 표시
- 기타 자유응답이 존재하면 데이터 테이블에서 `응답 보기` 버튼을 노출할 수 있다.
- 모달에서는 중복 제거 없이 실제 응답 텍스트 목록을 보여준다.
- 기타 응답은 차트 표시보다 상세 확인 보조 기능에 가깝게 다룬다.

## 7. 예외 및 비지원 처리

### 데이터 부족
- 문항은 지원 유형이더라도 실제 집계에 필요한 컬럼이 없으면 결과를 그리지 못할 수 있다.
- 이 경우 **§0 비지원 문항 유형**과 동일한 안내를 보여준다.

### 순위 문항 컬럼 불완전
- 순위형은 순위별 expanded 컬럼을 찾지 못하면 정상 집계를 수행하지 않는다.
- raw 값만 있다고 해서 순위 차트를 강제로 생성하지 않는다.

### 중복 문항 컬럼 불완전
- 중복형은 보기별 expanded 이진 컬럼을 찾지 못하면 정상 집계를 수행하지 않는다.

## 8. 문서 운영 원칙

- 문서보다 실제 구현이 우선이다.
- `dashboard.html`, `visualizations.js` 동작이 바뀌면 이 문서도 함께 갱신한다.
- 문서에는 기획안이 아니라 현재 화면에서 실제로 보이는 규칙을 적는다.
- 구현에서 `design_system.md` 토큰 목록에 없는 임의 색상·크기 값을 사용하게 되면 이 문서의 **1. 디자인 토큰** 섹션을 먼저 갱신하고 사용자에게 확인을 받는다.
