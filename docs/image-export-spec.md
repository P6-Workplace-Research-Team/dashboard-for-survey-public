# 차트 이미지 추출 기능 명세

## 1. 목적

대시보드에서 생성된 차트를 고객사 제출용 이미지로 추출한다.
내부 분석용 UI 컨트롤(보기 방식 전환, 정렬 옵션 등)은 이미지에서 제외하고,
차트 본문과 문항 정보만 포함한 클린한 이미지를 출력한다.

---

## 2. 출력 사양

| 항목 | 값 |
|---|---|
| 해상도 | 1920 × 1080 px |
| 비율 | 16:9 |
| 포맷 | PNG |
| 배경 | 흰색 (`#ffffff`) |
| 파일명 | 단일 페이지: `{문항 제목}.png` / 복수 페이지: `{문항 제목}_1.png`, `{문항 제목}_2.png`, … |

---

## 3. 추출 트리거

- 각 `result-section`마다 **"이미지로 저장" 버튼**을 개별 배치한다.
- 버튼 위치: `.result-header` 영역 우측
- 버튼은 이미지에 포함되지 않는다 (캡처 전 일시 숨김 처리).

---

## 4. 이미지에 포함할 요소

| 요소 | CSS 클래스 / 설명 |
|---|---|
| 문항 제목 | `.result-title` |
| 문항 전체 텍스트 | `.result-question-full` (Q. 본문 텍스트) |
| 차트 본체 | `.single-hbar-chart`, `.vbar-chart`, `.pie-chart-wrap`, `.stacked-chart-wrap`, `.lollipop-chart` 등 차트 영역 전체 |
| 가중 평균 설명 주석 | `.result-note` (산출식 설명 텍스트) |
| 범례 | `.result-side-panel`, `.result-legend` 등 범례 영역 (그룹 비교 적용 시 그룹 범례 포함) |
| 데이터 테이블 | `[data-data-table-section]` — **펼쳐진 상태인 경우에만** 포함 |

---

## 5. 이미지에서 제외할 요소

내부 분석 도구에 해당하는 UI 컨트롤은 캡처 직전 일시적으로 숨기고, 캡처 완료 후 복원한다.

| 요소 | CSS 클래스 / 속성 | 설명 |
|---|---|---|
| 그래프 유형 드롭다운 + 정렬 버튼 | `.chart-view-controls` | 가로/세로/원형/누적 전환, 기본순서/응답률 높은 순 |
| 순위 문항 컨트롤 전체 | `.choice-controls` | 가중 평균/응답 비율 탭, 막대 방향 탭, 가중 평균 높은 순서 정렬 체크박스 |
| 데이터 테이블 토글 버튼 | `[data-data-table-toggle]` | "데이터 테이블 펼치기/숨기기" 버튼 — 테이블 펼침 여부와 무관하게 항상 제외 |
| 이미지 저장 버튼 자체 | `.export-img-btn` | 캡처 버튼이 이미지에 찍히지 않도록 |

---

## 6. 레이아웃 처리

차트 섹션은 문항에 따라 높이가 가변적이므로, 1920×1080 고정 캔버스에 차트를 배치할 때 다음 규칙을 따른다.

1. 캡처 대상(`result-section`)의 렌더링 너비를 **1920px**로 고정한 상태에서 캡처한다.
2. 캡처된 차트 이미지를 1920×1080 흰 배경 캔버스에 **왼쪽 정렬**로 합성한다.
3. 차트 높이가 1080px 미만인 경우, 상하 여백을 동일하게 배분한다.
4. 차트 높이가 1080px를 초과하는 경우, **복수 페이지**로 분할 추출한다.
   - 분할 기준: 컴포넌트가 잘리지 않는 가장 아래쪽 경계를 기준으로 페이지를 나눈다.
   - 각 페이지는 1920×1080 흰 배경 캔버스에 해당 구간의 내용을 왼쪽 정렬로 합성한다.
   - 마지막 페이지의 남은 여백은 흰색으로 채운다.
   - 분할 단위 기준이 되는 컴포넌트 경계 후보 (우선순위 순):
     - 데이터 테이블 상단
     - 차트 본체 상단
     - 문항 질문 텍스트 상단
   - 파일명: `{문항 제목}_1.png`, `{문항 제목}_2.png`, …

---

## 7. 구현 방식

- **라이브러리**: `dom-to-image-more` (SVG 처리 정확도 우수)
- **폰트**: 로컬 `assets/fonts/Pretendard-*.otf` 참조 (CDN CORS 문제 회피)
- **캡처 흐름**:
  1. 제외 대상 요소(`[data-data-table-toggle]`, `.chart-view-controls`, `.choice-controls`, `.export-img-btn`)를 `visibility: hidden` 처리
  2. 데이터 테이블이 **접힌 상태**라면 `[data-data-table-section]`도 추가로 숨김
  3. `document.fonts.ready` 대기
  4. `domtoimage.toCanvas(section, { width: 1920, bgcolor: '#ffffff' })` 호출
  5. 반환된 캔버스 높이에 따라 단일/복수 페이지 분기:
     - 1080px 이하: 1920×1080 캔버스에 수직 중앙 합성 후 PNG 저장
     - 1080px 초과: 컴포넌트 경계 기준으로 구간 분할 → 각 구간을 1920×1080 캔버스에 합성 → 페이지 번호 붙여 복수 PNG 저장
  6. 제외했던 요소 복원
